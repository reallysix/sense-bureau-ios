版本：1.2

## 1. 使用方式

本文件是项目 UI 的视觉与交互单一事实来源。任何 SwiftUI 页面、组件、图表、主题和视觉资产开始实现前，必须先阅读本文件。

规范关键词：

- **必须**：实现不可偏离。
- **应该**：默认遵循，偏离时需要给出可验证理由。
- **禁止**：不得进入生产 UI。

参考图只用于提炼视觉语言，不复制其中的品牌、人物、运动数据、版式成品或第三方标识。

## 3. 设计定位

### 关键词

工业仪器、编辑式排版、信号感、机械刻度、真实数据、年轻但不幼稚。

### 核心原则

1. **先像可靠仪器，再像潮流产品。** 数据必须真实、清晰、可解释。
2. **用结构制造科技感。** 科技感来自刻度、数据、比例、响应和材质，而不是蓝紫渐变或发光边框。
3. **一个屏幕只有一个视觉主角。** 通常是当前读数、仪表或正在发生的扫描。
4. **装饰必须服务状态。** 每一条线、每一个脉冲、每一圈刻度都应对应测量或交互状态。
5. **主题换外观，不换事实。** 皮肤不能修改数值、单位、测量公式、警报含义和核心操作顺序。

### 明确禁止

- 蓝紫霓虹、无意义光晕、玻璃拟态和大面积渐变。
- 每个区域都做成相同的圆角白卡片。
- 用 emoji、Unicode 符号或临时图形充当正式图标和插画。
- 为了“看起来丰富”伪造数据、网格、扫描点或曲线。
- 在同一屏幕同时使用多个主强调色。
- 把点阵字体用于正文、按钮、设置项或长标题。
- 为不同皮肤复制整套页面代码。

## 4. 默认科技皮肤：Tech Signal

默认皮肤不是常见的赛博朋克，而是“黑色精密仪器 + 高可见信号橙 + 灰色数据显示屏”。

### 配色比例

- 普通页面：黑/深灰约 70%，灰色数据面约 20%，信号橙约 10%。
- 核心测量页面允许信号橙扩大到 35%–55%，但必须形成一个连续主区域，禁止零碎撒满全屏。
- 橙色背景上默认使用近黑文字；深色背景上使用暖白文字。
- “强磁场”是测量强度，不等同于危险，默认使用信号橙而不是红色。
- 状态不能只靠颜色表达，必须同时具有文字、图标或形态变化。

## 5. 字体系统

### 字体角色

| 角色 | 实现要求 | 使用位置 |
| --- | --- | --- |
| `displayDot` | 正式发布时必须内置具有合法授权的点阵数字字体 | 当前场强、峰值、角度、计时等主读数 |
| `displayNumeric` | SF Pro Display / SF Mono，使用等宽数字 | 点阵字体不支持的数值或无障碍回退 |
| `heading` | SF Pro Display Semibold | 页面标题、模块标题 |
| `labelInstrument` | SF Mono Medium/Semibold，大写并增加字距 | LIVE、CALIBRATE、PEAK、单位标签 |
| `body` | SF Pro Text Regular | 说明、引导、能力边界 |

### 点阵数字规范

- `displayDot` 是整个品牌的签名元素，必须出现在主实时读数中。
- 字体必须支持 `0–9`、小数点、负号、冒号；若缺少 `μ` 或单位字符，单位必须使用 `labelInstrument` 单独排版。
- 必须使用等宽数字或可预测的固定 advance，防止实时更新时布局横向抖动。
- 最小字号为 40pt；低于 40pt 时改用 `displayNumeric`，禁止把点阵字缩到不可读。
- 点阵数字最多承担一行主要信息；单位、状态和解释不得使用点阵字体。
- 实时更新不得使用滚轮数字动画；使用固定基线和 100–150ms 轻微交叉淡化。
- 最终字体文件进入项目之前必须记录名称、版本和许可；未确认许可时只能使用开发回退字体。

### 字号与层级

| 令牌 | 建议字号 | 行高 | 用途 |
| --- | --- | --- | --- |
| `readingHero` | 64–84pt | 0.90–0.96 | 单一核心读数 |
| `readingLarge` | 44–56pt | 1.0 | 次级仪表读数 |
| `metricLarge` | 28–36pt | 1.0 | 卡片数值 |
| `titlePage` | 20–24pt | 1.15 | 页面标题 |
| `titleSection` | 17–20pt | 1.2 | 区块标题 |
| `bodyPrimary` | 15–17pt | 1.35 | 正文和说明 |
| `label` | 11–13pt | 1.2 | 仪表标签、单位、日期 |

规则：

- 数字与单位必须分层，单位通常是数字字号的 22%–30%。
- 仪表标签优先大写，字距建议 0.08–0.14em。
- 正文禁止全大写和过宽字距。
- 必须支持 Dynamic Type；进入无障碍字号后允许重排，禁止整体缩小页面。

## 6. 网格、间距与形状

### 网格

- 使用 4pt 基础网格。
- 标准页面水平安全边距为 20pt；紧凑屏幕可降至 16pt，大屏可增至 24pt。
- 同级组件间距优先使用 8pt 或 12pt。
- 区块间距优先使用 24pt 或 32pt。
- 所有数值基线必须对齐；单位不应漂浮在不同高度。

### 间距令牌

| 令牌 | 值 |
| --- | --- |
| `space1` | 4pt |
| `space2` | 8pt |
| `space3` | 12pt |
| `space4` | 16pt |
| `space5` | 24pt |
| `space6` | 32pt |
| `space7` | 48pt |

### 圆角令牌

| 令牌 | 值 | 用途 |
| --- | --- | --- |
| `radiusSmall` | 6pt | 标签、迷你数据窗 |
| `radiusMedium` | 10pt | 标准仪表卡、按钮 |
| `radiusLarge` | 16pt | 大型容器 |
| `radiusNavigation` | 22pt | 底部导航 |
| `radiusPill` | 999pt | 状态胶囊，仅在语义确实为标签时使用 |

禁止把所有卡片统一做成 20pt 以上圆角。整体轮廓应接近工业设备面板，圆角只是防止生硬，不是主要风格。

### 边框和阴影

- 默认依靠色块、1pt 边框和层次区分区域，不依赖阴影。
- 重要仪表可使用 2pt 深色外框。
- 应用内部禁止大面积柔光阴影和发光轮廓。
- 按压状态通过颜色、位移 1pt 或缩放到 0.98 表达。

## 7. 页面构图

### 通用页面结构

1. 顶部安全区与紧凑仪表标题栏。
2. 单一主测量区，占首屏视觉权重的 45%–65%。
3. 关键辅助指标或实时曲线。
4. 次级工具、历史或操作区。
5. 固定底部导航或主操作区。

### 构图规则

- 可以使用非对称构图和局部裁切的大圆形仪表，但不能遮挡关键数值。
- 主要数值应在用户进入页面后 1 秒内被看到。
- 同一屏幕最多一个大型圆形仪表；其他指标使用小卡片或横向行。
- 内容密度可以高，但必须通过对齐、色块和字号层级分组，不能依靠大量留白掩盖结构问题。
- 首屏必须让用户同时知道“正在测什么”“当前数值”“单位”“当前状态”。

## 8. 核心组件规范

### 8.1 仪表标题栏

- 中间为页面/模块标题，使用 `labelInstrument` 或 `titleSection`。
- 左右可以放日期、状态或模式标签，高度 28–32pt，圆角 6pt。
- 不使用大面积透明毛玻璃导航栏。

### 8.2 主读数区

- 左上角显示测量名称，例如 `MAGNETIC FIELD`。
- 中部使用 `displayDot` 显示主数值。
- 单位单独排版并与数值基线对齐。
- 必须同时展示 `LIVE / CALIBRATING / PAUSED / UNSUPPORTED` 等状态。
- 数值变化不能引起卡片宽度、单位位置或其他控件抖动。

### 8.3 环形仪表

- 适用于磁场强度、倾角、校准进度和方向信息。
- 刻度数量建议 36、48 或 60，主刻度与次刻度必须有层级。
- 圆心区域只放一个核心数值和一个单位/状态。
- 指针或扫描扇区必须对应真实数据；禁止纯装饰性随机旋转。

### 8.4 实时曲线

- 数据屏优先使用 `surfaceData`，曲线使用近黑或暖白，峰值使用信号橙。
- 线宽 1.5–2pt；网格线 1pt、低对比度。
- 原始数据不做视觉上的过度平滑；若展示滤波数据，必须与产品逻辑一致。
- 至少标出当前值、峰值和时间范围，不显示没有含义的坐标。

### 8.5 工具列表卡

- 建议高度 68–84pt，圆角 10pt。
- 左侧为 40–48pt 图标/状态区，中间为工具名和主要指标，右侧可放迷你曲线或状态。
- 卡片不能只靠图标表达功能，必须保留文字名称。

### 8.6 按钮

- 最小可点击区域 44×44pt。
- 主按钮使用信号橙底、近黑文字；次按钮使用描边或深灰底。
- 标准按钮圆角 8–10pt，不使用渐变和悬浮发光。
- 测量进行中禁止出现会遮挡实时读数的全宽浮动按钮。

### 8.7 底部导航

- 高度 64–72pt，深色一体化容器，圆角 22pt。
- 3–5 个项目；当前项目使用信号橙，其他项目使用 `textSecondary`。
- 使用正式图标库或项目资产，禁止 emoji。
- 标签可以在空间允许时显示；VoiceOver 名称始终必须存在。

### 8.8 主题选择器

- 必须展示真实缩略预览，不只显示主题名称或颜色圆点。
- 切换后保持当前模块、测量会话、校准和历史数据不变。
- 主题选择本地持久化。

### 8.9 首次传感器引导

- 必须在传感器启动和首次校准前出现，先说明能力边界，再给出环境与配件准备步骤。
- 内容区可以滚动，主操作固定在底部；小屏上必须为固定操作预留足够滚动余量。
- 必须提供“跳过”和“开始校准”，两者都要保存完成状态；用户可从设置重新打开引导。
- 主题只能改变表面、图标和语气，不能删减“可以检测 / 不能识别”以及干扰来源等事实。
- 引导页禁止使用伪造扫描、成功发现或安全结论来制造效果。

## 9. 数据表达

- 数值必须带单位；不使用无依据的小数位。
- 默认磁场主读数使用整数 `μT`，只有验证确有价值时才展示一位小数。
- 强度等级使用“低 / 中 / 强”或同等中性表述，不使用“安全 / 危险”代替未经验证的结论。
- 校准必须显示进度和原因，而不是只有旋转加载图标。
- 无数据、传感器不可用和权限不足是不同状态，必须分别设计。
- 历史曲线与实时曲线必须在标题或样式上可区分。

## 10. 动效、触觉与声音

### 科技皮肤动效

- 普通状态过渡：150–220ms，`easeOut`。
- 主题切换：180–240ms 交叉淡化；禁止重建页面造成闪白或数据中断。
- 仪表指针：使用物理合理的平滑插值，不追求无意义弹性。
- 扫描动画：1.2–1.8 秒一轮，仅在真实测量进行时出现。
- 按压：80–120ms，缩放至 0.98 或下移 1pt。

### 触觉与声音

- 轻触操作使用轻触觉；校准完成使用成功触觉；超过用户阈值使用可节流的提醒触觉。
- 连续信号不得每帧触发声音或触觉，必须设置节流和静音开关。
- 科技皮肤声音应短、干净、偏机械，不使用游戏胜利音效。
- 系统静音、用户关闭声音和 Reduce Motion 设置必须被尊重。

## 11. 可换皮肤架构

### 共享不变量

所有皮肤必须共享：

- 页面信息架构和导航位置。
- 测量公式、采样、滤波和阈值语义。
- 数值、单位、状态和错误文案的含义。
- 最小触控尺寸、动态字体、VoiceOver 和 Reduce Motion。
- 主读数的点阵数字品牌特征；若卡通皮肤需要变化，只能改变点形、颜色或轻微动效，不能降低可读性。

### 允许主题覆盖

- 颜色语义令牌的具体值。
- 图标、插画、背景材质和非数据装饰资产。
- 圆角在限定范围内的变化。
- 动效曲线、提示音和触觉配置。
- 文案语气，但不能改变事实或能力边界。

### 卡通皮肤边界

- 卡通皮肤不是给科技皮肤加贴纸；它应拥有独立的配色、正式插画资产和反馈语言。
- 卡通插画后续必须通过独立视觉方案确定，不在没有选定视觉目标时临时绘制。
- 禁止使用 emoji、CSS/SwiftUI 几何拼图或随意手绘符号冒充最终角色和图标。
- 卡通皮肤可以更圆润、更有弹性，但必须保持数据密度和仪器可信度。

### SwiftUI 实现约束

- View 只能引用语义令牌，例如 `theme.colors.canvasPrimary`，禁止在业务 View 中直接写十六进制色值。
- 字体、间距、圆角、描边、动效、声音和触觉都必须通过 ThemeKit 提供。
- Feature View 只维护一份；主题通过 Environment 注入。
- 主题资源必须具备完整性检查，缺少必需令牌或资产时构建测试应失败。

## 12. 无障碍与可读性

- 正文和关键文字对比度目标不低于 4.5:1，大号文字不低于 3:1。
- 点阵数字只用于大字号；VoiceOver 必须以自然语言读出数值和单位。
- 实时数值不应持续打断 VoiceOver；默认由用户聚焦后读取，重要阈值事件才主动提示。
- 不以颜色作为唯一状态信号。
- 支持 Dynamic Type、Bold Text、Reduce Motion、Increase Contrast 和系统静音。
- 横屏不是首版核心，但页面不得因较短高度裁掉校准或停止测量控件。

## 13. 资产规范

- 正式图标优先使用一致的系统/授权图标库；缺少时生成或设计真实资产。
- 插画、纹理、声音和自定义字体必须保留来源与许可信息。
- 不从参考图中裁切第三方 UI、图标、人物或品牌素材作为产品资产。
- 位图资产按实际槽位生成，不拉伸、不低清放大。
- 所有主题资产需同时提供浅/深背景适配或明确固定背景。

## 14. 设计验收矩阵

每个新页面至少验证：

| 维度 | 必测状态 |
| --- | --- |
| 设备尺寸 | 标准 Pro 尺寸、Pro Max 大屏、最小支持宽度 |
| 测量状态 | 初始、校准中、实时、暂停、峰值、无数据、不可用 |
| 主题 | 科技皮肤；卡通皮肤发布后加入同一矩阵 |
| 字体 | 默认、较大、无障碍字号 |
| 系统设置 | Reduce Motion、Increase Contrast、静音 |
| 内容长度 | 中英文、长单位、较大数值、负值和小数 |

视觉完成标准：

1. 真实 App 已运行并截取相关尺寸和状态。
2. 实现截图与参考图/已批准设计目标并排检查。
3. 修复可见的边距、裁切、字体、层级、对比度、圆角和状态问题。
4. 再次截图验证；仅通过构建、单元测试或 Preview 不算视觉完成。

## 15. 开发前检查清单

- [ ] 已阅读本设计规范和相关功能计划。
- [ ] 已确认页面唯一视觉主角。
- [ ] 已使用 ThemeKit 语义令牌，没有业务 View 硬编码样式。
- [ ] 主实时读数使用点阵字体角色，单位使用普通仪表字体。
- [ ] 页面不存在蓝紫霓虹、玻璃拟态、无意义渐变或发光。
- [ ] 所有交互区域至少 44×44pt。
- [ ] 数据状态、无数据、不可用和错误状态均有设计。
- [ ] 已验证 Dynamic Type、VoiceOver 和 Reduce Motion。
- [ ] 已完成真实运行截图与视觉复核。

## 16. 尚未阻塞开发的待定项

- 点阵字体的最终字体文件、授权和字形覆盖范围。
- 卡通皮肤的角色、插画和完整配色方向。
- App 图标和启动画面。

这些项目必须在对应资产进入生产版本前确定，但不阻塞 ThemeKit、科技皮肤和传感器样机开发。
