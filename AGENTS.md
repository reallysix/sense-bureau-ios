# Project Instructions

## UI and visual work

- Before creating or changing any UI, read `DESIGN_SYSTEM.md` completely and treat it as the project's visual source of truth.
- Use the source images in `design/references/` only according to the reference hierarchy documented in `DESIGN_SYSTEM.md`.
- Do not introduce styles that conflict with the design tokens, typography roles, component anatomy, theme contract, or prohibited-pattern list in the design system.
- For every UI change, run the global `ui-completion-verification` skill before declaring completion. A build or test pass alone is not visual verification.
- If a requested UI change conflicts with `DESIGN_SYSTEM.md`, surface the conflict before implementation instead of silently overriding the specification.
