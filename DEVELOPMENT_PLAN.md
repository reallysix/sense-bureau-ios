# 可换皮肤的 iPhone 随身实验室：可行性评估与开发计划

更新日期：2026-07-22

UI 实现必须遵循：[DESIGN_SYSTEM.md](DESIGN_SYSTEM.md)

正式品牌已确认为“感知局 / Sense Bureau”，公开仓库名为 `sense-bureau-ios`。完整命名依据和工程约定见：[BRAND_DECISIONS.md](BRAND_DECISIONS.md)。

## 1. 结论

项目技术上可行，建议继续，但要调整产品表达和首版范围：

- 产品应定位为“可换皮肤的 iPhone 随身实验室”或“磁场探测器”，不能把手机能力宣传成可识别所有金属的专业金属探测仪。
- “换皮肤”值得做，但应是统一主题系统，不是为科技版、卡通版分别开发两套页面。
- 首版先用磁场、振动、水平/倾角、气压/高度四个成熟模块验证需求；LiDAR、声音频谱、NFC、UWB 放到后续。
- 默认科技风，同时免费提供一套完整卡通风，用真实用户数据验证年轻用户是否更喜欢、是否更愿意分享，而不是先假定“年轻人一定不喜欢科技风”。
- 技术栈首选 SwiftUI 原生。当前工作区为空，应按全新 iOS 项目估算。

综合可行性：

| 维度 | 评估 | 说明 |
| --- | --- | --- |
| 传感器能力 | 高 | Core Motion 可读取磁力计、加速度计、陀螺仪等数据 |
| 主题切换 | 高 | SwiftUI 的环境值和语义化设计令牌适合全局主题切换 |
| 首版开发风险 | 中 | 实机校准、MagSafe 干扰、实时曲线性能需要验证 |
| 产品差异化 | 中高 | 工具类 App 常见，但“严谨测量 + 年轻化主题”有传播差异 |
| 商业可行性 | 中 | 需要验证留存和付费；仅靠皮肤不足以支撑订阅 |
| 审核风险 | 中 | 主要风险来自夸大“金属探测”能力和不准确的商店文案 |

建议立项条件：先完成约 2–4 小时的传感器技术验证和约 4–6 小时的双主题原型，再根据实机数据进入完整 MVP。真实目标用户测试仍需预留外部反馈时间。

## 2. 对共享文档的评估

### 判断正确的部分

1. iPhone 可以读取三轴磁场数据。Core Motion 提供磁力计原始数据，轴值单位为微特斯拉；可计算场强 `sqrt(x² + y² + z²)` 并展示变化。
2. 它本质上是磁场异常探测，不是专业金属探测。对磁铁、铁、钢和电机附近的异常更敏感，对铜、铝、金、银等非铁磁金属不应作可靠承诺。
3. 原生 SwiftUI 比 Flutter 更适合首版。核心能力高度依赖 Core Motion、AVFoundation、Accelerate、ARKit、RoomPlan、Core NFC、Nearby Interaction 和 StoreKit；使用 Flutter 仍需维护 Swift 桥接层。
4. 第一版无需服务器。测量、校准、历史记录、主题设置和买断权益都可以本地完成。
5. UWB 不能扫描任意物体。Nearby Interaction 要求兼容设备或配件参与并交换必要信息。

### 需要收紧的部分

1. 文档中的探测距离只能作为经验描述，不能成为营销承诺；不同机型、手机壳、环境磁场和目标物都会造成明显差异。
2. “年轻人不适合科技风”目前只是产品假设。应测试科技、卡通两种风格在 18–24 岁人群中的首选率、理解度、分享意愿和次日回访。
3. 文档给出的价格是参考，不是已验证结论。首发价格应在 TestFlight 或小范围上线后根据转化数据确定。
4. “功能越多受众越广”不一定成立。首版模块过多会稀释质量，尤其换皮肤会放大视觉和测试工作量。
5. App Store 文案不能暗示探测黄金、墙内电线、地下硬币或危险物；Apple 明确反对虚假功能和误导性营销。

## 3. 产品定义

### 目标用户

- 核心：18–24 岁，对数码、探索、桌面设备和轻量实验感兴趣的用户。
- 次级：数码博主、维修爱好者、学生、摄影和家装场景用户。

### 核心价值

把 iPhone 已有传感器变成容易理解、愿意反复打开、适合分享的轻量工具；同一测量能力可以在“专业仪表”和“卡通探索”两种表达之间切换。

### 产品命名原则

- 首页和功能名优先使用“磁场探测”“磁场雷达”“附近磁场异常”。
- “金属探测”只能作为用户易懂的场景解释，同时显著说明能力边界。
- 不使用“探测所有金属”“找黄金”“找墙内电线”等不可验证表述。

### 两套首发皮肤

| 维度 | 科技皮肤（默认） | 卡通皮肤 |
| --- | --- | --- |
| 气质 | 专业、冷静、仪表感 | 轻松、探索、收藏感 |
| 视觉 | 深色、高对比、网格/波形 | 明亮色、圆润容器、角色化反馈 |
| 图表 | 标准曲线、刻度、峰值 | 保留同一数据，换成更友好的图形外观 |
| 动效 | 扫描、脉冲、平滑过渡 | 弹性反馈、角色状态、奖励动画 |
| 音效 | 简洁蜂鸣、雷达提示 | 轻量提示音、发现反馈 |
| 文案 | “校准”“阈值”“峰值” | “先找一块安静区域”“发现异常” |

两套皮肤必须共享以下不变量：信息架构、实时数据、单位、警报等级、操作步骤、触控区域、权限解释、错误提示和测量逻辑。

## 4. MVP 范围

### 必做

1. 首次使用引导
   - 解释磁场探测的真实能力和限制。
   - 提醒移除磁吸壳、MagSafe 配件和充电宝。
   - 引导在无目标物的环境中完成基准校准。

2. 工具首页
   - 磁场探测、振动分析、水平/倾角、气压/高度四个入口。
   - 展示设备是否支持对应传感器。

3. 磁场探测
   - 三轴原始值、综合场强、相对基准变化值。
   - 5–10 秒基准校准。
   - 平滑滤波、峰值记录、可调阈值。
   - 实时曲线、声音和触觉提示开关。
   - 明确的“可检测/不可可靠检测”说明。

4. 振动分析
   - 实时加速度、强度趋势、峰值和短时记录。

5. 水平/倾角
   - 水平状态、X/Y 角度、归零。

6. 气压/高度
   - 相对气压和相对高度变化，不把相对值包装成专业海拔仪。

7. 主题中心
   - 科技、卡通两套完整主题预览和一键切换。
   - 切换后不停止测量、不重置数据。
   - 主题选择本地持久化。
   - 支持“减少动态效果”、关闭音效/触觉和动态字体。

8. 本地记录
   - 保存时间、模块、摘要和峰值。
   - 首版提供最近记录，不做账户和云同步。

9. 设置与隐私
   - 权限状态、单位、主题、音效、触觉、数据清除、能力边界和隐私说明。

### 设置模块优先级

| 优先级 | 设置项 | 排序理由与上线条件 |
| --- | --- | --- |
| P0 | 简体中文 / English、能力边界说明 | 直接影响用户是否看懂测量结果；首批实现，选择本地保存并即时生效 |
| P1 | 音效、触觉、警报阈值、单位 | 实现成本低，但必须在对应声音、触觉和测量逻辑已接入后提供，禁止空开关 |
| P1 | 主题中心 | 属于产品核心差异化；两套正式主题资源齐备后上线，切换不得中断测量 |
| P2 | 权限状态、数据清除、隐私详情 | 随记录和权限流程完善后上线；数据清除必须二次确认 |
| P2 | 其他语言 | 复用现有本地化结构，依据目标市场和用户反馈逐步加入 |

### 明确不做

- LiDAR 房间扫描、NFC、UWB、声音频谱、频闪检测。
- AI 结果解释、云同步、账户、社区和排行榜。
- Flutter/Android 版本。
- 广告、订阅和大量皮肤商店。
- 外接 USB-C/蓝牙硬件。

## 5. 推荐技术架构

### 技术栈

- UI：SwiftUI
- 传感器：Core Motion
- 图表：Swift Charts
- 本地数据：SwiftData
- 购买能力（后续）：StoreKit 2
- 测试：XCTest、XCUITest、视图快照测试、实机传感器测试

### 分层

```text
AppShell
├── ThemeKit
│   ├── ThemeDefinition
│   ├── SemanticColors / Typography / Spacing
│   ├── Icons / Illustrations / Sounds / Haptics
│   └── ThemeManager
├── MeasurementCore
│   ├── SensorClient
│   ├── CalibrationEngine
│   ├── FilterPipeline
│   ├── ThresholdEngine
│   └── RecordingStore
└── Features
    ├── MagneticField
    ├── Vibration
    ├── Level
    └── Barometer
```

### 主题系统约束

- 所有视觉实现以 `DESIGN_SYSTEM.md` 为强制基线；本节只描述架构，不重复视觉规则。
- 页面只能引用语义令牌，例如 `surfacePrimary`、`readingNormal`、`readingWarning`，不能直接散落颜色值。
- 主题可以更换色彩、字体组合、图标、插画、背景材质、非关键动效、提示音和触觉配置。
- 主题不能更改测量公式、阈值含义、单位、危险提示和核心操作顺序。
- 不为不同主题复制 Feature View；只维护一套页面结构。
- 每个新组件必须同时通过两套主题、动态字体和减少动态效果检查。

### 实时数据策略

- 传感器采样与 UI 刷新解耦；采样频率按模块需要设置，UI 以较低频率合并更新。
- 磁场模块同时保留原始值、滤波值和基准差值，便于调试和解释异常。
- App 进入后台或页面离开时停止无用的实时更新，避免耗电和发热。
- 所有传感器能力先检查硬件可用性，再显示功能入口或降级说明。

## 6. Codex 执行周期

本项目由 Codex 持续完成产品拆解、编码、自动化测试、构建、模拟器验证和视觉检查，不需要按传统团队的人员排期、任务交接和会议节奏估算。

当前环境已经具备 Xcode 26.6、Swift 6.3.3 和 iOS 模拟器，可立即开始。需要注意：模拟器不能替代真实磁力计、气压计和设备干扰测试，因此 iPhone 17 Pro Max 的连接、签名和实物测试仍需要用户侧配合。

### 总体时间

| 交付层级 | Codex 有效执行时间 | 预计日历时间 | 不包含 |
| --- | --- | --- | --- |
| 磁场技术样机 | 2–4 小时 | 当天 | 实物重复测试时间 |
| 双主题可交互演示 | 累计 6–10 小时 | 1–2 天 | 真实目标用户招募 |
| 四工具完整 MVP | 累计 24–40 小时 | 3–5 个连续工作日 | TestFlight 观察期 |
| TestFlight 发布候选版 | 累计 32–50 小时 | 4–7 天 | Apple 审核等待 |
| 有真实反馈验证的首发版 | 额外修复 4–12 小时 | 10–16 个日历日 | App Store 审核不确定性 |

这里的“有效执行时间”指我实际进行代码、测试和修复的时间；如果真机、证书、测试账号或反馈不能及时提供，日历时间会相应顺延，但不会增加核心开发工作量。

### 阶段 0：产品和技术验证（约 2–4 小时）

交付：仅在实机运行的磁力计技术样机，不做完整 UI。

- 建立 Xcode 工程、Git 和基础 CI。
- 读取 x/y/z、综合场强和设备可用性。
- 完成基准校准、简单滤波、阈值和日志导出。
- 用磁铁、钢制螺丝、铁片、铜/铝物体、MagSafe 配件进行重复测试。
- 记录不同朝向、手机壳和环境下的数据。

通过标准：

- 在稳定环境中连续运行 10 分钟无崩溃、无明显发热。
- 对选定铁磁目标在约定近距离内重复测试，成功触发率达到 90% 以上。
- 稳定环境误报警低于每分钟 1 次，并能通过校准或阈值控制。
- 团队接受它是“磁场异常检测”而不是“全金属识别”。

未通过则停止扩大范围，先调整算法或重新定义产品。

### 阶段 1：双主题产品原型（约 4–6 小时）

交付：磁场主界面的科技和卡通两套高保真、可切换交互原型。

- 固定相同的信息架构和数据内容。
- 为两套主题定义语义令牌、图标/插画方向、动效、音效和文案语气。
- 完成可在模拟器和真机切换的两套高保真界面。
- 原型完成后，再找 8–12 名 18–24 岁目标用户完成任务测试；这部分是外部验证时间，不计入 Codex 开发时间。
- 记录首选主题、首次测量成功率、能力理解度、分享意愿和卡顿点。

通过标准：

- 80% 以上用户能正确复述“不能探测所有金属”。
- 80% 以上用户无需指导完成校准并开始测量。
- 至少一套非默认皮肤对目标用户产生清晰偏好或更高分享意愿；否则减少主题投入，仅保留基础外观选项。

当前进度（2026-07-20）：工程原型已完成。科技信号与趣味实验室两套主题共享同一磁场页面和测量会话，支持设置页真实读数预览、一键切换和本地持久化；切换不会重建 `MagneticFieldViewModel`。模拟器自动化与视觉检查已通过。8–12 名目标用户任务测试及真机重复测量仍属于本阶段未完成的外部验收，不应据此提前宣称阶段 1 的产品假设已经验证。

### 阶段 2：基础架构（约 4–6 小时）

- 完成 AppShell、导航、设置和主题管理。
- 完成 MeasurementCore、传感器生命周期和本地数据模型。
- 建立科技/卡通主题完整性测试和关键算法单元测试。

验收：运行时切换主题不重启页面、不停止采样、不丢失记录。

当前进度（2026-07-20）：工程开发已完成。AppShell 已提供实验室、磁场、设置三段导航，首页只把已实现的磁场模块标为可用；共享测量会话会在离开页面或 App 进入后台时停止传感器，返回后恢复采样并保留校准、暂停状态和已有样本。SwiftData 最近记录模型与读写存储已经建立，两套主题的必需语义令牌由完整性测试约束。14 个单元测试与 4 个 UI 测试全部通过，并已在 iPhone SE 小屏和 iPhone 17 Pro 标准尺寸检查科技/卡通、中英文界面。真实磁力计的长时运行、后台恢复与干扰测试仍并入阶段 4 的真机矩阵，不由模拟器结果替代。

### 阶段 3：四个 MVP 模块（约 12–20 小时）

- 批次 A：磁场探测完整功能与引导。
- 批次 B：振动分析、水平/倾角。
- 批次 C：气压/高度、最近记录、设置和错误状态。

每个模块都要覆盖正常、校准中、无传感器、权限/系统限制、暂停、记录完成等状态，并在两套主题下验收。

当前进度（2026-07-22）：批次 A、B、C 的模拟器开发与自动化验收均已完成。磁场模块包含首次使用引导、能力边界与干扰提示、三轴原始值、综合场强、相对变化、可调阈值、声音/触觉单次越界提醒和 SwiftData 测量保存；振动模块提供实时强度、均方根、峰值、3 秒趋势、基准校准、暂停恢复和记录保存；水平/倾角模块提供总倾角、X/Y 角度、真实二维气泡映射、归零、暂停恢复和记录保存。气压/高度模块现已提供气压、气压变化、相对高度、会话基准、趋势曲线、暂停恢复和记录保存，并在传感器缺失、权限拒绝和运行失败时给出不同状态。实验室首页与完整记录页可查看最近测量，设置支持 hPa/kPa 单位切换、本机数据说明和二次确认清除。四个模块统一处理前后台及导航生命周期，并明确标注模拟器演示数据。26 个单元测试和 12 个 UI 测试全部通过，已检查 iPhone SE、iPhone 17 Pro 和 iPhone 17 Pro Max 上的科技/卡通、中英文关键状态。阶段 3 的软件实现完成；磁场、振动、姿态和气压计精度、提示音/触觉体感及长时运行仍需在阶段 4 使用带签名的真机构建验证，模拟器结果不能替代该验收。

### 阶段 4：质量和 TestFlight（约 8–12 小时开发验证 + 7 天观察）

- 算法单元测试、传感器实机矩阵测试、UI 自动化和快照测试。
- iPhone 17 Pro Max 为主设备，并至少增加两款不同代际的真机。
- 检查动态字体、VoiceOver、对比度、减少动态效果、静音和触觉关闭。
- 检查 15–30 分钟连续测量的耗电、温度、前后台切换和来电中断恢复。
- 生成 TestFlight 候选构建后，邀请 20–30 名用户使用至少 7 天；7 天是观察窗口，不是开发工期。

发布门槛：

- 关键任务完成率不低于 90%。
- 测量中切换主题的数据连续性为 100%。
- 没有 P0/P1 缺陷，测试期无重复崩溃问题。
- 商店文案、截图和引导没有夸大探测能力。

### 阶段 5：首发准备（约 3–5 小时，Apple 审核时间另计）

- 完成 App Store 页面、隐私清单、审核备注、支持页面和演示说明。
- 默认科技主题，卡通主题免费开放，用于验证主题价值。
- 首版不做订阅；根据留存、主题切换率和用户访谈决定后续商业化。

## 7. 商业化计划

### 首发

- 免费提供四个基础工具、科技和卡通两套主题。
- 先收集主题选择率、7 日留存、测量次数、记录使用率和分享行为。
- 不在测量过程中插入广告。

### 数据验证后

优先尝试“免费 + Pro 终身买断”：

- 免费：基础测量、最近记录、两套首发主题。
- Pro：无限历史、数据对比、CSV/图片/PDF 导出、高级阈值和专业模块。
- 额外主题包可作为独立非消耗型内购，但不能让基础可读性依赖付费皮肤。
- 只有出现云同步、AI 分析、持续模板和团队协作等持续价值后，才考虑订阅。

App 内解锁数字功能和皮肤应使用 Apple 允许的 In-App Purchase 方案。

## 8. 关键指标

### 产品指标

- 首次测量完成率。
- 首次启动到获得有效读数的时间。
- 主题预览率、切换率、最终选择率。
- 每周测量次数、次日和 7 日留存。
- 记录保存、导出和分享使用率。
- 用户对“磁场探测边界”的理解率。

### 技术指标

- 传感器会话启动成功率。
- 稳定场景误报警率、目标物重复触发率。
- 主题切换导致的数据丢失次数（目标为 0）。
- 连续 30 分钟运行的崩溃、内存、耗电和温升表现。
- 两套主题的页面覆盖率和快照差异审核完成率。

## 9. 主要风险和应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| “金属探测仪”预期过高 | 差评、退款、审核风险 | 改为磁场探测定位，首屏和商店页明确边界 |
| MagSafe 和手机自身磁场干扰 | 误报警、数据漂移 | 移除磁吸配件提示、每次会话校准、相对变化值 |
| 卡通皮肤影响测量可信度 | 用户觉得是玩具 | 关键读数与单位保持一致，卡通仅改变表达层 |
| 两套 UI 分叉 | 开发和测试成本翻倍 | 单一 View + 语义令牌 + 主题资源包 |
| 实时曲线造成耗电和卡顿 | 体验差、发热 | 采样与绘制解耦，限频刷新，后台停止更新 |
| 年轻化假设不成立 | 差异化无效 | 双主题原型完成后立即做用户验证，未通过就缩小主题投入 |
| 首版功能过多 | 延期、质量下降 | 固定四个 Core Motion/气压模块，其余排入后续 |
| 商业化过早 | 低评分、转化差 | 首发验证留存，后续再上买断和主题包 |

## 10. 后续路线

- V1.1：声音频谱、数据导出、Pro 买断。
- V1.2：LiDAR 测距/房间能力；只在支持设备展示。
- V1.3：NFC 工具和兼容配件能力。
- V2：AI 解释、云同步、设备数据库、外接硬件和 Studio 订阅。

每增加一个模块，都必须先证明真实硬件能力，再把它接入统一测量、记录、主题和导出框架。
